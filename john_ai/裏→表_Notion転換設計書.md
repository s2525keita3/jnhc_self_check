# 裏（Googleドライブ）→ 表（Notion）へ流す設計書

コンサルアドバイザーが裏で貯めたナレッジを、既存Notionの公開DBへ転換するための「変換マップ」。
**この設計に沿って、AI（Claude）が匿名化・整形してNotionに書き込む。**

> 原則：個社が特定される情報（社名・実数・個人名）は **必ず匿名化・一般化** してからNotionへ。
> 例：「合同会社Size」→「ある開業準備中のステーション」、「2000万円」→「約2千万円規模」。

---

## 全体像

```
Googleドライブ「コンサルアドバイザー/」          Notion（既存・公開）
─────────────────────              ──────────────
各社コンサルログ（Sheet）の                      ┌─ Q&A・よくあるご質問 DB
 「ナレッジ候補・メモ」列        ─抽出・匿名化─▶  ├─ 🎬 コンテンツライブラリ DB
コンサル_ナレッジベース（Sheet）                  └─ 提供資料・テンプレート DB
```

## 運用フロー（月次 or ナレッジが溜まったとき）

1. **拾う**：ナレッジベースSheet（`1ENLM-w9Hxnyd9JViprZQjd0LyBGUqyONGX8fsaVIc2A`）で `転用候補` が付いた行、各社ログの「ナレッジ候補・メモ」を見る。
2. **判定**：QA→①、ノウハウ/失敗/型→②、テンプレ→③ に振り分け。
3. **匿名化**：社名・実数・個人名を一般化。
4. **書き込み**：上の変換マップに沿って該当NotionDBへ1件作成。
5. **記録**：ナレッジベースSheetの該当行の `ステータス` を「Notion公開済」に更新（＝二重投稿防止）。

## SNS・起業塾・協会への展開（Notionの先）

Notionの「コンテンツライブラリ」に入った記事は、さらに二次展開できる：
- **SNS投稿** … 記事の要点を3〜5投稿に割る（フック→本文→CTA）
- **起業塾カリキュラム** … ノウハウ＋失敗点をセットでレッスン化
- **協会教材** … マニュアル・フレームワークを体系化
これらは必要時に、Notion記事 or ドライブのナレッジを元にAIが原稿化する。

---

## 実行時のNotion書き込み仕様（AI向けメモ）

- ページ作成は Notion MCP の `notion-create-pages` を使い、`parent`に上記data-source URLを指定。
- select/multi_selectは**既存の選択肢名と完全一致**させる（新規作成しない）。
- 公開前に1件ずつ内容を提示し、承認を得てから書き込む（外向き変更のため）。
