# john_ai

john_ai は、コンサルティング業務で生まれるあらゆる情報を「記憶・整理・分析・再利用」するための
ナレッジ運用システム（セカンドブレイン）です。

AI（Claude / john_ai）が継続的にこのリポジトリを読み書きし、
クライアント支援とコンテンツ・商品化を回し続けることを前提に設計しています。

---

## このリポジトリで実現すること

1. **記憶する** — クライアントとのZoom会議・議事録・課題・やり取りを失わず蓄積する
2. **整理する** — 現状・論点・タスクを構造化し、いつでも「今どうなっているか」が分かる状態にする
3. **分析する** — PDCAで仮説検証を回し、何が効いて何が効かなかったかを残す
4. **抽出する** — 全セッションから再利用可能なノウハウ・マニュアル・QA・失敗点を取り出す
5. **再利用する** — 蓄積した資産をポータル/SNS/協会/起業塾/商品へ転換し続ける

```
インプット            蓄積・整理              抽出・分析            アウトプット
─────────         ───────────         ──────────         ─────────────
Zoom会議    ──▶   クライアント/    ──▶   ナレッジベース/  ──▶   ポータルサイト
議事録              （個社の記憶）          （横断の知見）         SNS投稿
課題                                      PDCA分析              協会コンテンツ
QA                                       失敗点・教訓          起業塾
                          ▲                                    商品化
                          └──────── PDCAで戻して改善 ──────────┘
```

## フォルダ構成（トップ）

```
john_ai/
├── README.md                ← このファイル（全体像）
└── コンサルティング/          ← 本システムの本体
    ├── README.md             運用ルールの入口（まずここを読む）
    ├── _システム/             運用マニュアル・ルール・テンプレート
    ├── クライアント/           クライアントごとの記憶
    ├── ナレッジベース/         全セッション横断の知見（ノウハウ/QA/失敗点…）
    └── アウトプット/           二次活用・商品化（ポータル/SNS/協会/起業塾/商品）
```

詳細は [`コンサルティング/README.md`](コンサルティング/README.md) を参照。

## AI（john_ai）への基本指示

新しい会話・セッションを始めるときは、必ず次の順で読み込むこと。

1. `コンサルティング/_システム/運用マニュアル.md` — 全体の動かし方
2. 対象クライアントがあれば `コンサルティング/クライアント/<会社名>/00_プロフィール.md` と `05_タイムライン.md`
3. 関連する `コンサルティング/ナレッジベース/_INDEX.md`

セッション終了時は、必ず「記録 → 抽出 → 反映」の3ステップを実行する（運用マニュアル参照）。
