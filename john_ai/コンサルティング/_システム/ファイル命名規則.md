# ファイル命名規則

一貫した命名で、AIも人間も検索・整列・再利用しやすくする。

---

## 1. 共通ルール

- **日付は `YYYY-MM-DD`**（例：`2026-05-30`）。先頭に置くと自動で時系列ソートされる。
- 単語の区切りは **アンダースコア `_`**。スペースは使わない。
- 日本語ファイル名OK。ただし `/ \ : * ? " < > |` は使わない。
- 連番は **2桁ゼロ埋め**（`01` `02` …）。

---

## 2. フォルダ・ファイルの命名

| 対象 | 形式 | 例 |
|---|---|---|
| クライアントフォルダ | `<会社名 or 屋号>` | `株式会社山田製作所` |
| プロフィール | `00_プロフィール.md` | （固定） |
| タイムライン | `05_タイムライン.md` | （固定） |
| 議事録 | `YYYY-MM-DD_<トピック>.md` | `2026-05-30_キックオフ.md` |
| 課題 | `課題一覧.md`（1ファイルで管理） | （固定） |
| PDCA | `YYYY-MM-DD_<施策名>.md` | `2026-05-30_初回面談台本改訂.md` |
| 資料・成果物 | `YYYY-MM-DD_<種別>_<名前>.<拡張子>` | `2026-05-30_提案書_集客改善.pdf` |
| ナレッジ | `<種別略号>-<連番>_<短い名前>.md` | `KH-001_紹介が増える仕組み.md` |

### ナレッジの種別略号
- `KH` = ノウハウ（Know-How）
- `MN` = マニュアル
- `QA` = QA
- `FL` = 失敗点・教訓（Failure / Lesson）
- `FW` = フレームワーク

---

## 3. プレフィックスの意味

- `_` 始まり（例：`_システム/` `_INDEX.md` `_example_…`）
  = 「索引・設定・サンプル」など、通常のコンテンツと区別したいもの。一覧の先頭に並ぶ。
- `00_`〜`05_` = クライアント内の標準フォルダ/ファイルの並び順固定用。

---

## 4. ステータス表記（ファイル先頭メタで統一）

- 課題：`未着手` / `対応中` / `保留` / `完了`
- PDCA：`計画中` / `実行中` / `評価中` / `完了`
- クライアント：`見込み` / `アクティブ` / `休眠` / `終了`

---

## 5. ファイル先頭メタ（推奨フォーマット）

各 `.md` の冒頭に、機械可読なメタブロックを置くと検索・集計しやすい。

```
---
種別: ノウハウ
タグ: #集客 #士業
作成日: 2026-05-30
更新日: 2026-05-30
元ネタ: 株式会社サンプル / 2026-05-30 セッション   ← 公開時は削除
転用候補: SNS投稿, 起業塾Lesson
---
```
