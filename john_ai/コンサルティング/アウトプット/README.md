# アウトプット（二次活用・商品化）

`ナレッジベース/` に貯まった知の資産を、収益・影響力につながる形に転換するレイヤー。

> 🔒 **公開の鉄則**：ここに置くものは外に出る前提。クライアントが特定できる情報は
> 必ず匿名化・一般化してから持ち込む（運用マニュアル6章）。

---

## 転換マップ（ナレッジ → アウトプット）

| ナレッジの種類 | 向いている行き先 |
|---|---|
| ノウハウ（KH） | SNS投稿 / ポータル記事 / 起業塾レッスン |
| マニュアル（MN） | 協会教材 / 起業塾カリキュラム / 商品（テンプレ販売） |
| フレームワーク（FW） | ワークシート商品 / 有料note / 協会教材 |
| QA | ポータルFAQ / チャットボット / SNSの小ネタ |
| 失敗点・教訓（FL） | SNS（共感系）/ 起業塾の注意事項 / コラム |

---

## フォルダと役割

```
アウトプット/
├── ポータルサイト/    記事・FAQ・事例（匿名）。Web資産の元データ
├── SNS投稿/          X/Instagram/LinkedIn等の投稿ネタ・原稿・カレンダー
├── 協会コンテンツ/    協会の教材・認定講座・会員向けコンテンツ
├── 起業塾/           カリキュラム・レッスン・課題・受講生向け資料
└── 商品化/           有料商品（教材/テンプレ/講座）の企画・LP草案・価格設計
```

---

## ワークフロー（月次が基本）

1. `ナレッジベース/_INDEX.md` で `転用候補` の付いたエントリを抽出する。
2. 行き先のフォルダに、そのフォーマット（記事/投稿/レッスン等）へ書き起こす。
3. **匿名化チェック**を行う（実名・数字・特定情報を消す）。
4. 公開・配信し、反応（反応数・申込・売上）を記録する。
5. 反応が良かったものは `ナレッジベース/` に「刺さった表現」としてフィードバック。

---

## コンテンツ・パイプライン（再掲）

```
個社の出来事 → ナレッジ抽出 → 匿名化 → アウトプット形式に変換 → 公開/販売 → 反応を計測 → ナレッジに還元
```

1つのナレッジを**複数の出口に展開**するのが基本（1ノウハウ → SNS投稿＋記事＋レッスン）。
