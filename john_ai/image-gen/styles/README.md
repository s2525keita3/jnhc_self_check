# styles / トンマナプリセット

ここにブランドのトンマナ(tone & manner)プリセットを Markdown で置く。
エージェント(Codex)は画像を作る前にここを読み、プロンプトに反映させる。

## 使い方

1. 案件・媒体ごとに `<name>.md` を作る(例: `sales-slide.md`, `youtube-thumb.md`)。
2. 1ファイル = 1トンマナ。下の項目を埋める。
3. 画像生成を頼むときに「`styles/youtube-thumb.md` のトンマナで」と指定すれば、
   エージェントがその内容をプロンプトへ織り込む。

## テンプレート

各プリセットファイルは次の構成で書くと、エージェントが拾いやすい:

```markdown
# <プリセット名>

## 用途
（例: YouTube サムネ / 研修スライド / セールス資料）

## カラーパレット
- メイン: #RRGGBB
- アクセント: #RRGGBB
- 背景: #RRGGBB

## フォント / 文字の雰囲気
（太ゴシック / 手書き風 / serif など。日本語可否も）

## トーン・雰囲気
（例: 信頼感・清潔・プロフェッショナル / 明るくポップ）

## 構図のルール
（被写体の位置、余白、文字を入れる/入れない 等）

## やってほしくないこと（NG）
（例: 過度な装飾、ストックフォト感、特定色 など）

## 参考イメージ
（あれば参考画像のパスや説明）
```

> トンマナは後日改めて支給される想定。届いたらこのフォルダに追加する。
